# DreamStudio v1 PRD 确认版

本文档是 DreamStudio v1 的产品需求确认版，基于 `01-dreamstudio-v1-product-shape.md` 和第二阶段作答结果整理。

当前版本状态：确认版。

本确认版用于指导后续架构设计、数据模型设计、接口契约和开发里程碑。产品范围如需调整，应先更新本文档，并同步清理或更新被替代的旧文档。

---

## 1. 产品摘要

### 1.1 产品定位

DreamStudio v1 是一个依赖用户自有 `new-api` 密钥的一站式 AI 创作平台。

用户先在 `new-api` 中注册、充值并创建 API 密钥，再在 DreamStudio 中注册登录并填写密钥。DreamStudio 不处理模型成本、充值、订阅和支付，只负责提供更易用的 AI 创作体验、异步任务、资产保存和管理后台。

### 1.2 v1 主场景

v1 只聚焦 AI 绘画。

视频生成、AI 对话、订阅、支付、订单、团队空间、社区、分享、模板市场等能力不进入 v1。

### 1.3 目标用户

v1 目标用户：

- 已经拥有 `new-api` 账号和密钥的普通用户。
- 希望通过浏览器直接进行 AI 绘画的内容创作者。
- 不想安装和配置第三方客户端的 AI 重度用户。

### 1.4 核心价值

- 降低 `new-api` 用户使用 AI 绘画能力的门槛。
- 通过浏览器提供即开即用的创作体验。
- 让生图任务在服务端持续执行，避免用户关闭页面导致任务丢失。
- 保存参考图、生成结果、任务记录和请求日志。
- 让管理员可以配置模型、存储、任务超时、重试和用户状态。

## 2. 产品边界

### 2.1 DreamStudio v1 负责

- DreamStudio 用户注册与登录。
- DreamStudio 用户启用与禁用。
- 用户 `new-api` 服务地址和 API 密钥配置。
- 用户 `new-api` 连接测试，优先使用 `/v1/models`，避免测试调用消耗额度。
- AI 绘画创作台。
- 图片模型选择。
- 生图参数设置。
- 参考图上传。
- 异步生图任务。
- 任务状态展示。
- 失败提示和重试。
- 结果图保存。
- 资产仓库。
- 请求日志。
- 管理后台。
- 本地存储和 S3 兼容存储配置。

### 2.2 DreamStudio v1 不负责

- `new-api` 账号注册。
- `new-api` 密钥创建。
- `new-api` 充值。
- 用户支付。
- 用户订阅。
- 模型成本计算。
- 渠道管理。
- 倍率配置。
- 订单管理。
- 发票和退款。
- 视频生成。
- AI 对话。
- 团队协作。
- 分享链接。

### 2.3 new-api 负责

- 模型统一接口。
- 上游模型厂商适配。
- 用户在 `new-api` 中的额度。
- 用户在 `new-api` 中的充值。
- 用户在 `new-api` 中的密钥。
- 上游模型调用、失败和成本。

## 3. 角色与权限

### 3.1 普通用户

普通用户可以：

- 注册和登录 DreamStudio。
- 配置自己的 `new-api` 服务地址和 API 密钥。
- 测试自己的 `new-api` 配置。
- 使用 AI 创作台提交生图任务。
- 查看自己的任务。
- 查看、下载、删除自己的生成结果。
- 查看、删除自己的上传参考图。
- 修改自己的密码。

普通用户不能：

- 访问其他用户的数据。
- 访问管理后台。
- 查看系统存储配置。
- 查看请求日志全量数据。
- 查看其他用户的 `new-api` 配置。

### 3.2 超级管理员

超级管理员可以：

- 登录管理后台。
- 查看用户列表。
- 启用或禁用用户。
- 代用户配置或重置 `new-api` 密钥。
- 查看用户密钥状态。
- 配置默认 `new-api` 服务地址。
- 配置是否允许用户自定义 `new-api` 服务地址。
- 管理模型目录，固定模型类型为聊天、图片、视频。
- 管理模型推荐状态、图标、描述和端点能力。
- 配置本地存储。
- 配置 S3 兼容存储。
- 配置任务超时时间。
- 配置任务重试次数。
- 查看请求日志。

超级管理员不能：

- 查看已保存的用户 `new-api` 密钥明文。
- 在日志中查看完整 Authorization Header。

管理员代用户配置或重置密钥时，系统必须记录审计日志。

## 4. 用户主流程

### 4.1 首次使用流程

1. 用户访问 DreamStudio 首页。
2. 用户点击开始使用。
3. 系统跳转登录/注册页。
4. 用户注册 DreamStudio 账号。
5. 用户登录 DreamStudio。
6. 系统检测用户未配置 `new-api` 密钥。
7. 系统展示 `new-api` 密钥配置引导。
8. 用户按引导前往 `new-api` 获取密钥。
9. 用户返回 DreamStudio 填写密钥。
10. 如果管理员允许自定义服务地址，用户填写 `new-api` 服务地址。
11. 如果管理员不允许自定义服务地址，用户只填写密钥，系统使用默认服务地址。
12. 系统优先通过 `/v1/models` 测试连接。
13. 测试通过后进入 AI 创作台。
14. 测试失败时保留配置并显示修正提示。

### 4.2 AI 绘画流程

1. 用户进入 AI 创作台。
2. 用户选择图片模型。
3. 用户填写提示词。
4. 用户按需上传参考图。
5. 用户设置尺寸、比例、分辨率、生成张数等参数。
6. 用户点击生成。
7. 系统创建异步任务。
8. Worker 使用用户密钥调用 `new-api`。
9. 前端展示任务状态。
10. 任务成功后展示结果图。
11. 系统将结果图保存到资产仓库。
12. 用户可下载、查看或删除结果图。

### 4.3 任务失败流程

1. Worker 调用 `new-api` 失败。
2. 系统记录请求日志和任务错误摘要。
3. 如果错误可重试，系统按配置自动重试。
4. 如果重试耗尽，任务进入失败状态。
5. 前端展示可读错误提示。
6. 用户可手动重新提交任务。

### 4.3.1 任务取消流程

1. 用户取消尚未开始执行的 `pending` 任务。
2. 系统将任务标记为 `canceled`。
3. Worker 不再执行该任务。
4. 对已经进入 `running` 的任务，v1 返回不可取消，不强制要求真正取消上游请求。

### 4.4 额度不足流程

1. `new-api` 返回疑似额度不足错误。
2. DreamStudio 将任务标记为失败。
3. 前端提示用户前往 `new-api` 检查余额或充值。
4. DreamStudio 不处理充值。

### 4.5 密钥失效流程

1. `new-api` 返回认证失败或服务不可用。
2. DreamStudio 将用户配置标记为异常。
3. 前端提示用户检查服务地址和密钥。
4. 用户进入个人中心重新配置。
5. 系统重新测试连接。

## 5. 页面需求

### 5.1 首页

目标：

- 表达 DreamStudio 是一站式 AI 创作平台。
- 引导用户点击开始使用。
- 不在首屏直接强调“必须准备 new-api 密钥”。

功能：

- 展示产品名称。
- 展示核心价值。
- 提供登录入口。
- 提供开始使用入口。

验收标准：

- 未登录用户点击开始使用后进入登录/注册页。
- 已登录用户点击开始使用后进入 AI 创作台或密钥配置引导。
- 已登录用户访问公开首页时，右上角应显示用户头像入口，能直接进入下一步页面。

### 5.2 登录/注册页

功能：

- 用户名密码注册。
- 用户名密码登录。
- 登录错误提示。
- 注册错误提示。
- 登录态默认 30 天，可通过 Redis/session 或 refresh token 延续。

v1 不做：

- 邮箱验证。
- 手机验证码。
- OAuth 登录。
- 2FA。

验收标准：

- 用户可以完成注册并登录。
- 被禁用用户不能登录。
- 登录成功后跳转到下一步所需页面。
- 已登录用户再次打开登录或注册页时，应自动跳回下一步所需页面。

### 5.3 密钥配置引导

触发条件：

- 用户首次登录后未配置 `new-api` 密钥。
- 用户密钥测试失败。
- 用户主动进入个人中心修改配置。

功能：

- 填写 API 密钥。
- 根据管理员配置决定是否显示服务地址输入框。
- 展示默认服务地址说明。
- 测试连接。
- 保存配置。
- API 密钥以掩码方式展示。

验收标准：

- 管理员关闭自定义服务地址时，普通用户看不到服务地址输入框。
- 管理员开启自定义服务地址时，普通用户可以填写服务地址。
- 密钥保存后不展示明文。
- 测试失败时给出明确提示。

### 5.4 AI 创作台

功能：

- 展示固定筛选：全部、聊天、图片、视频、我的。
- 展示管理员启用的模型，并支持按名称、模型 ID、描述快速搜索。
- 选择模型。
- 输入提示词。
- 上传参考图。
- 按模型级参数 Schema 配置生图参数。
- 提交生图任务。
- 展示任务状态。
- 展示生成结果。
- 在创作台左上角任务弹层查看当前模型最近图片任务。

v1 常见参数：

- 尺寸。
- 比例。
- 分辨率。
- 生成张数。
- 参考图。
- 负面提示词。
- Seed。
- 输出格式。
- 图片质量。

参数规则：

- 每个模型通过参数 Schema 决定支持哪些参数。
- 基础参数默认展示，高级参数默认折叠。
- 切换模型时保留通用参数，清空或禁用目标模型不兼容的参数。

验收标准：

- 未配置有效密钥的用户不能提交任务。
- 被禁用模型不能被普通用户选择。
- 用户提交任务后页面能看到任务状态。
- 切换模型后创作台任务弹层只展示当前模型任务。
- 用户关闭页面后任务继续执行。
- 模型不支持的参数不能继续提交。

### 5.5 任务页

功能：

- 查看当前用户任务列表。
- 查看任务状态。
- 查看任务创建时间。
- 查看任务模型。
- 查看任务结果。
- 查看失败摘要。
- 手动重新提交失败任务。
- 取消还未开始执行的 `pending` 任务。

任务状态：

- `pending`。
- `running`。
- `succeeded`。
- `failed`。
- `timeout`。
- `canceled`。

验收标准：

- 用户只能看到自己的任务。
- 任务状态刷新后与后端一致。
- 失败任务能展示可读错误摘要。
- 用户至少可以取消尚未开始执行的 `pending` 任务，已经 `running` 的任务 v1 不支持取消。

### 5.6 资产仓库

功能：

- 查看生成结果图。
- 下载图片。
- 删除图片。
- 批量删除图片。
- 按任务或创建时间筛选生成结果。

说明：

- v1 资产仓库只展示结果图。
- 上传参考图仍保存为资产，用于任务执行和任务详情展示，但不在资产仓库中作为独立列表展示。

v1 不做：

- 参考图资产列表。
- 文件夹。
- 标签。
- 搜索。
- 分享链接。
- 回收站。

验收标准：

- 用户只能看到自己的资产。
- 删除后不可恢复。
- 批量删除需要二次确认。

### 5.7 个人中心

功能：

- 查看账号信息。
- 修改密码。
- 查看密钥配置状态。
- 配置 `new-api` API 密钥。
- 测试 `new-api` 连接。

验收标准：

- 密钥不展示明文。
- 用户可自行重置密钥。
- 连接测试结果可见。

### 5.8 管理后台

管理后台页面：

- 用户管理。
- 模型管理。
- 系统设置。
- 存储设置。
- 请求日志。
- 审计日志。

用户管理功能：

- 查看用户列表。
- 启用用户。
- 禁用用户。
- 软删除用户。
- 重置用户密码。
- 代用户配置或重置密钥。

模型管理功能：

- 从 `new-api` 拉取候选模型。
- 管理聊天、图片、视频模型。
- 模型类型固定，不再由管理员维护分类。
- 管理模型是否启用。
- 管理模型是否推荐。
- 管理模型展示名称。

系统设置功能：

- 默认 `new-api` 服务地址。
- 是否允许用户自定义服务地址。
- 生图任务超时时间。
- 生图任务最大重试次数。
- 生图任务重试间隔。

存储设置功能：

- 本地存储。
- S3 兼容存储。
- 参考图保存策略。
- 结果图保存策略。
- 文件保留时间，单位为小时。

请求日志功能：

- 查看请求用户。
- 查看请求类型。
- 查看模型名称。
- 查看任务 ID。
- 查看请求状态。
- 查看错误摘要。
- 查看响应耗时。
- 查看 prompt 摘要。
- 在权限和审计约束下查看完整 prompt 与参数快照。

审计日志功能：

- 记录管理员代配置密钥。
- 记录管理员重置密钥。
- 记录存储配置变更。
- 记录用户启用和禁用。

## 6. 模型管理需求

### 6.1 模型来源

模型候选列表可通过 `new-api` 接口获取，但 DreamStudio 不直接将所有模型暴露给用户。

管理员需要手动维护：

- 哪些模型属于聊天、图片或视频。
- 哪些模型启用。
- 哪些模型推荐。
- 模型图标、展示名称和描述。
- 模型使用的端点类型，一个模型可同时支持多个端点。
- 用户收藏模型用于 Studio 的“我的”筛选。

### 6.2 v1 接口类型

v1 第一验收路径：

- `/v1/images/generations`。
- `/v1/images/edits`。

v1 后段兼容目标：

- `/v1beta/models/{model}:generateContent`。

说明：

- `/v1/images/generations` 用于文生图。
- `/v1/images/edits` 用于参考图、图生图或编辑场景。
- Gemini 风格接口进入 v1 范围，但不作为第一验收路径。

### 6.3 模型配置字段

每个模型建议配置：

- 模型 ID。
- 展示名称。
- 模型类型：聊天、图片或视频。
- 模型图标。
- 模型描述。
- 端点类型，可多选。
- 是否启用。
- 是否推荐。
- 是否支持参考图。
- 参考图传递方式：multipart 或 URL。
- 默认尺寸。
- 支持尺寸列表。
- 默认生成张数。
- 最大生成张数。
- 参数 Schema。

## 7. 非功能需求

### 7.1 安全

- 用户密钥必须加密保存。
- 密钥页面只显示掩码。
- 日志不得记录完整密钥。
- 管理员不能查看已保存密钥明文。
- 管理员代配置或重置密钥必须记录审计日志。
- 普通用户只能访问自己的任务和资产。
- 请求日志默认展示 prompt 摘要和脱敏参数。
- 管理员查看完整 prompt 或完整参数需要权限，并记录审计。

### 7.2 可靠性

- 生图任务必须异步执行。
- 用户关闭页面后任务继续执行。
- 用户至少可以取消尚未开始执行的 `pending` 任务。
- Worker 重启后任务不能无声丢失。
- 任务超时和重试由系统配置控制。
- 失败需要记录错误摘要。

### 7.3 存储

- 支持本地存储。
- 支持 S3 兼容存储。
- 支持参考图和结果图分开配置保留时间。
- 文件保留时间单位为小时。
- 删除后不可恢复。

### 7.4 部署

- 必须支持 Docker Compose。
- DreamStudio 以单应用容器部署，容器内部承载 Web、API 和 Worker 逻辑模块。
- PostgreSQL 和 Redis 可以使用 Docker Compose 本地服务，也可以使用云 PostgreSQL 和云 Redis。
- DreamStudio 与 `new-api` 分开部署。
- 保留基础健康检查和容器日志。
- v1 不强制内置自动备份，备份方案仅作为部署文档中的可选建议，不作为 v1 强制验收项。

## 8. v1 验收标准

v1 达到以下标准后可认为主流程跑通：

- 新用户可以注册、登录、配置密钥并通过连接测试。
- 连接测试优先使用 `/v1/models`，不依赖聊天接口。
- 管理员可以配置默认 `new-api` 服务地址和是否允许用户自定义服务地址。
- 管理员可以配置至少一个可用图片模型。
- 管理员可以为模型配置参数 Schema。
- 普通用户可以使用该模型提交生图任务。
- 生图任务通过 Worker 异步执行。
- 用户关闭页面后任务仍会继续。
- 用户可以取消尚未开始执行的 `pending` 任务。
- 任务成功后结果图保存到资产仓库。
- 用户可以查看、下载、删除自己的结果图。
- 任务失败时用户能看到可读错误提示。
- 管理员可以查看请求日志。
- 管理员不能查看用户已保存密钥明文。
- Docker Compose 可以启动 DreamStudio 单应用容器，并支持连接本地或云 PostgreSQL/Redis。
- 部署文档包含 PostgreSQL 和上传文件可选备份建议，但备份不作为 v1 强制验收项。

## 9. 架构阶段待决项

以下问题不影响 v1 产品范围确认，但会影响架构文档、数据模型和接口契约：

- 请求日志保留多久。
- 审计日志保留多久。
- 密钥加密主密钥如何配置和轮换。
- 哪些错误允许自动重试。
