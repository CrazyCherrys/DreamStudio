# DreamStudio v1 UI 视觉方向与组件规范

本文档基于 `06-dreamstudio-v1-ia-and-roadmap.md` 编写，用于指导 DreamStudio v1 的视觉风格、页面布局、组件规范、交互状态和前端实现。本文档不替代最终高保真设计稿，但应作为开发初版 UI 的基准。

当前版本状态：确认版。

当前默认视觉主题：暗色体系。代码中的 `apps/web/src/app/globals.css` 已将全站默认 token 切换为近黑背景、暗玻璃面板、青绿色主操作/选中态和琥珀强调态；浅米色主题只作为历史设计方向，不再是当前实现基准。

第七阶段的目标是让 DreamStudio v1 从一开始就具备可信、完整、有创作气质的产品体验，而不是临时后台模板或通用 AI 套壳。

---

## 1. 视觉定位

### 1.1 产品气质

DreamStudio v1 的视觉方向：

```text
专业创作工具 + 暗色工作室 + 轻量未来感
```

关键词：

- 专业。
- 安静。
- 清晰。
- 有创造力。
- 有质感。
- 不压迫。
- 不花哨。

避免：

- 通用 SaaS 后台模板感。
- 过度科技蓝。
- 默认紫白 AI 风格。
- 无层级的纯黑暗黑模式。
- 玻璃拟态堆叠。
- 卡片过多导致信息碎片化。

### 1.2 设计目标

v1 UI 需要做到：

- 首次访问时，用户相信这是一个可以认真使用的 AI 创作平台。
- 创作台让用户专注 prompt、模型、参数和结果。
- 管理后台清晰、克制、可靠，不追求花哨。
- 密钥配置流程要降低焦虑，明确告诉用户下一步做什么。
- 任务状态要可感知，避免用户怀疑任务是否丢失。

## 2. 品牌与色彩

### 2.1 色彩方向

确认使用“近黑中性色 + 青绿色能量色 + 琥珀强调色”。

当前实现默认是暗色主题，不再以浅米色、奶油色或白色面板作为默认视觉。页面可以保留温暖工作室气质，但应通过深色中性色、低透明度玻璃面板、琥珀强调和克制纹理表达。

主背景：

- `--ds-bg: #080a09`：全站近黑背景。
- `--ds-bg-soft: #101614`：页面柔和暗背景。
- `--ds-surface: #121816`：默认暗面板。
- `--ds-surface-raised: #17201d`：抬高面板、弹窗、表格行。
- `--ds-surface-muted: #0d1211`：低强调信息块和占位区域。

主品牌色：

- `--ds-brand: #2dd4bf`：主要按钮、选中状态、可执行动作。
- `--ds-brand-hover: #5eead4`：按钮 hover 和高亮文本。
- `--ds-brand-soft: rgba(45, 212, 191, 0.14)`：选中卡片和轻量提示底色。

辅助强调色：

- `--ds-accent: #f6b33d`：生成中、提示、关键强调。
- `--ds-accent-soft: rgba(246, 179, 61, 0.14)`：琥珀轻提示底色。
- `--ds-danger: #fb7185`：危险操作、失败状态。
- `--ds-success: #34d399`：成功状态。

文字和边框：

- `--ds-text: #f4f7f2`：主文字。
- `--ds-text-muted: #9daaa1`：辅助文字。
- `--ds-border: rgba(151, 176, 164, 0.18)`：默认边框。
- `--ds-border-strong: rgba(151, 176, 164, 0.34)`：强调边框、hover 边框。

### 2.2 CSS 变量建议

```css
:root {
  --ds-bg: #080a09;
  --ds-bg-soft: #101614;
  --ds-surface: #121816;
  --ds-surface-raised: #17201d;
  --ds-surface-muted: #0d1211;
  --ds-border: rgba(151, 176, 164, 0.18);
  --ds-border-strong: rgba(151, 176, 164, 0.34);
  --ds-text: #f4f7f2;
  --ds-text-muted: #9daaa1;
  --ds-brand: #2dd4bf;
  --ds-brand-hover: #5eead4;
  --ds-brand-soft: rgba(45, 212, 191, 0.14);
  --ds-accent: #f6b33d;
  --ds-accent-soft: rgba(246, 179, 61, 0.14);
  --ds-danger: #fb7185;
  --ds-success: #34d399;
  --ds-admin-sidebar: rgba(9, 13, 12, 0.92);
  --ds-admin-content: rgba(8, 10, 9, 0.72);
  --ds-shadow: 0 18px 50px rgba(0, 0, 0, 0.36);
}
```

规则：

- v1 默认使用暗色主题。
- 新页面应优先复用 `--ds-*` token、`.ds-card`、`.ds-button`、`.ds-input`、`.ds-badge` 和共享暗色 surface 工具类。
- 不应新增 `bg-white/*`、奶油色、浅米色或白底 `pre` 作为默认页面表面。
- 色彩表达要克制，主界面以深中性色、边框和暗玻璃层级为主。
- 生成按钮和关键状态使用青绿色，不要整站大面积铺满品牌色。
- 琥珀色只用于运行中、提示和关键强调，不作为大面积背景。

## 3. 字体与排版

### 3.1 字体建议

英文字体：

- 标题：`Fraunces` 或 `Newsreader`。
- 正文和界面：`IBM Plex Sans` 或 `Geist`。

中文字体：

- `Noto Serif SC` 可用于首页大标题或品牌语。
- `Noto Sans SC` 可用于界面正文。

说明：

- 不使用 Arial、Roboto、Inter 作为唯一默认视觉。
- 如果工程引入字体成本过高，可优先使用 `Noto Sans SC` + `IBM Plex Sans`。
- 首页标题可以更有性格，后台和创作台保持高可读性。

### 3.2 字号层级

建议层级：

| 用途 | 桌面字号 | 移动字号 |
| --- | --- | --- |
| 首页主标题 | 64px | 40px |
| 页面标题 | 32px | 28px |
| 区块标题 | 20px | 18px |
| 正文 | 15px | 15px |
| 辅助说明 | 13px | 13px |
| 表格文本 | 14px | 14px |

规则：

- 创作台和后台不追求大字号冲击，优先信息密度和清晰度。
- 首页可以使用更大胆的标题排版。
- 表单说明文字要短，不要长篇教育用户。

## 4. 布局系统

### 4.1 页面宽度

建议：

- 首页内容最大宽度：`1200px`。
- 登录注册页表单宽度：`420px`。
- 用户设置页内容宽度：`880px`。
- 管理后台内容最大宽度：`1280px`。
- 创作台桌面端使用全屏工作区。

### 4.2 间距

基础间距：

- `4px` 为最小单位。
- 常用间距：`8px`、`12px`、`16px`、`24px`、`32px`、`48px`。

规则：

- 表单字段之间建议 `16px`。
- 页面区块之间建议 `32px`。
- 管理后台表格密度适中，不要过度稀疏。

### 4.3 圆角与阴影

建议：

- 小控件圆角：`10px`。
- 卡片圆角：`18px`。
- 首页展示卡片圆角：`28px`。
- 弹窗圆角：`20px`。

阴影：

- 工作区内减少重阴影。
- 首页和浮层可使用柔和大阴影。
- 管理后台优先使用边框区分层级。

## 5. 首页设计规范

### 5.1 首页目标

首页 v1 不做临时轻量 Landing，需要具备完整产品表达和可信视觉完成度。

首页要表达：

- DreamStudio 是一个浏览器里的 AI 创作工作室。
- 用户可以通过自己的 `new-api` 密钥使用模型能力。
- DreamStudio 负责更好的创作体验、任务执行和资产管理。
- 用户不需要安装第三方客户端。

### 5.2 首页结构

建议模块：

1. 顶部导航。
2. Hero 主视觉。
3. 创作台能力展示。
4. 使用流程说明。
5. 任务和资产管理能力说明。
6. 管理员可配置能力说明。
7. 底部 CTA。

### 5.3 Hero

Hero 内容：

- 主标题：强调“一站式 AI 图片创作工作室”。
- 副标题：说明对接用户自己的 `new-api` 密钥。
- 主按钮：开始创作。
- 次按钮：登录。
- 轻提示：使用前需要准备 `new-api` API 密钥。

视觉建议：

- 使用近黑到深青绿色的克制暗色渐变背景。
- 使用抽象画布、生成结果网格、任务队列卡片构成主视觉。
- 不直接使用厂商 Logo 堆砌。
- 不用夸张赛博风。

主视觉结论：

- v1 首页采用“暗色工作室 + 抽象画布 + 任务卡片”的主视觉方向。

### 5.4 首页背景

建议：

- 近黑、深青绿色和极低透明度琥珀光感组成柔和暗色渐变。
- 叠加极淡网格或噪点纹理，但不使用浅米色纸张底。
- 使用 1 到 2 个低亮度抽象光感，不要满屏光污染。

### 5.5 首页动效

推荐：

- 首屏标题和卡片轻微错位进入。
- 任务卡片状态可做轻微脉冲。
- 结果图网格可做慢速浮动。

避免：

- 大量滚动视差。
- 复杂 3D。
- 动效影响表单可用性。

## 6. 登录、注册与密钥引导

### 6.1 登录注册页

视觉目标：

- 安静。
- 可信。
- 不像临时后台登录框。

布局：

- 左侧品牌说明或创作插画。
- 右侧登录/注册表单。
- 移动端只保留表单和简短品牌语。
- 表单容器使用暗玻璃 `.ds-card`，输入框使用暗色 `.ds-input`。

状态：

- 表单错误使用明确文案。
- 登录中按钮展示 loading。
- 被禁用用户展示独立提示。

### 6.2 密钥配置引导

密钥配置是用户的第一个关键门槛，需要尽量降低焦虑。

页面结构：

- 标题：连接你的 `new-api` 密钥。
- 三步说明：前往 `new-api`、创建密钥、粘贴到 DreamStudio。
- 默认服务地址展示。
- API 密钥输入。
- 测试连接。
- 保存并进入创作台。

文案规则：

- 明确说明 DreamStudio 不处理充值和额度。
- 明确说明密钥会加密保存。
- 不展示密钥明文。
- 测试失败时给出下一步建议。

状态设计：

- 未测试：中性提示。
- 测试中：按钮 loading，表单禁用。
- 测试成功：绿色成功状态。
- 测试失败：红色错误摘要 + 修复建议。

## 7. AI 创作台设计规范

### 7.1 桌面双区域布局

v1 AI 创作台采用：

```text
左侧模型栏 | 右侧统一创作容器
```

确认宽度：

- 左侧模型栏：`280px` 到 `320px`。
- 右侧统一创作容器：自适应，最小 `520px`。

布局规则：

- 左右区域高度占满视口。
- 左侧模型栏独立滚动。
- 右侧统一创作容器包含画布、结果预览、任务入口、任务弹层和底部 Prompt 子容器。
- 底部 Prompt 子容器属于统一创作容器，不使用明显顶部分割线或重阴影切成独立第三栏。
- 生成按钮放在底部 Prompt 子容器内，始终容易触达。
- `.studio-workspace` 使用独立但同源的暗色 token：更深背景、更沉侧栏、青绿色选中态、低亮度琥珀强调。
- Studio 内部不使用全局白色面板；高级参数表单中的共享组件也需要被暗色覆盖。

### 7.2 左侧模型栏

组件：

- 分类 tabs。
- 推荐模型区。
- 模型列表。
- 模型状态标签。

模型卡片展示：

- 展示名称。
- 厂商名称。
- 是否支持参考图。
- 推荐标签。
- 简短能力说明。

状态：

- 选中。
- hover。
- 禁用。
- 推荐。

### 7.3 统一创作容器

组件：

- 画布与结果预览区。
- 左上角“任务列表”琥珀胶囊按钮。
- 当前模型历史任务浮层。
- Prompt 编辑器。
- 负面提示词，高级区默认折叠。
- 参考图上传区。
- 参数表单。
- 生成按钮。

Prompt 编辑器：

- 支持多行输入。
- 底部一体化 Prompt 区可以使用紧凑输入高度，桌面端当前不低于 `82px`；独立编辑器或详情编辑场景不低于 `160px`。
- 提供字数或 token 近似提示，但不强制精确。
- 为空时生成按钮禁用。

参考图上传：

- 不支持参考图的模型隐藏或禁用上传区。
- 未上传时展示默认参考图占位。
- 单张上传后展示单图缩略图。
- 多张上传后在同一区域层叠展示，避免占用 Prompt 输入区域。
- 鼠标悬停或键盘聚焦堆叠区域时展开全部参考图，清晰露出每张预览。
- 每张缩略图右上角提供关闭按钮；关闭只从本次生成的参考图选择中移除，不删除资产库文件。
- 达到本次生成参考图上限后，继续上传入口禁用或提示上限。

参数表单：

- 基础参数直接展示。
- 高级参数折叠。
- 不兼容参数不展示。
- 使用模型 `parameter_schema` 驱动。

### 7.4 当前模型任务弹层

组件：

- 左上角“任务列表”胶囊按钮，使用琥珀强调色。
- 无边框感深色浮层。
- 搜索框。
- 当前模型名称。
- 已显示数量。
- 关闭按钮。
- 空状态、加载状态和错误状态。
- 任务行展示 Prompt 摘要、状态、`MM/DD HH:mm` 创建时间和详情链接。

状态展示：

- `pending`：等待中。
- `running`：生成中。
- `succeeded`：已完成。
- `failed`：失败。
- `timeout`：超时。
- `canceled`：已取消。

规则：

- `/studio` 的任务浮层只展示当前选中模型最近 50 条图片任务。
- 切换模型后重新加载当前模型历史任务。
- 有 `pending` 或 `running` 任务时定时刷新当前模型列表。
- 提交成功后立即将新任务插入当前模型历史列表。
- 完整任务管理、取消、重试和删除仍在 `/studio/tasks` 完成。
- 成功后画布优先展示结果图，失败时展示可读错误摘要。

### 7.5 移动端布局

移动端降级：

- 顶部模型选择。
- 中间 prompt 和参数。
- 统一创作容器内展示任务入口、画布和底部 Prompt 子容器。
- 模型列表可用抽屉。

规则：

- 移动端不强制双区域同屏。
- 任务弹层在移动端按容器全宽收缩，任务行允许换行，避免文字重叠。
- 生成按钮应始终容易触达。
- 任务结果不要被参数表单淹没。

## 8. 资产仓库设计规范

### 8.1 范围

v1 资产仓库只展示结果图。

不展示：

- 参考图列表。
- 文件夹。
- 标签。
- 分享链接。
- 回收站。

说明：

- 参考图仍保存为资产，用于任务执行和任务详情展示。
- 用户不在资产仓库中独立管理参考图。

### 8.2 布局

推荐：

- 顶部筛选区。
- 瀑布流或规则网格。
- 右侧或弹窗预览详情。

卡片展示：

- 图片缩略图。
- 模型名称。
- 创建时间。
- 下载按钮。
- 删除按钮。

批量选择：

- 进入选择模式后显示多选框。
- 批量删除前二次确认。

### 8.3 空状态

文案示例：

```text
还没有生成结果。
去创作台生成第一张作品。
```

空状态应提供进入 `/studio` 的按钮。

## 9. 管理后台设计规范

### 9.1 后台首页

v1 后台首页只做导航面板和关键配置入口。

模块：

- 用户管理入口。
- 模型管理入口。
- 系统设置入口。
- 存储设置入口。
- 请求日志入口。
- 审计日志入口。

不做：

- 复杂统计图表。
- 实时大屏。
- 成本分析。
- 收入分析。

### 9.2 后台整体风格

后台应：

- 信息密度适中。
- 表格清晰。
- 操作按钮明确。
- 危险操作需要二次确认。
- 以可靠为主，不追求强视觉冲击。
- 默认使用暗色后台 shell：侧栏为近黑半透明面板，内容区为暗色背景。
- 表格行、提示框、筛选块和 JSON/日志 `pre` 使用 `--ds-surface-raised`、`--ds-surface-muted` 或全局代码面板，不使用白色行底。

### 9.3 表格

表格规范：

- 每页默认 20 条。
- 支持筛选和分页。
- 行操作放在右侧。
- 危险操作使用红色或二次确认。

适用页面：

- 用户管理。
- 模型管理。
- 模型候选快照。
- 请求日志。
- 审计日志。

### 9.4 表单

表单规范：

- 字段标签清晰。
- 帮助文本短且具体。
- 保存按钮固定在表单底部或右上角。
- 保存成功有 toast。
- 保存失败展示错误摘要。

适用页面：

- 系统设置。
- 存储设置。
- 模型编辑。
- 用户密钥代配置。

## 10. 模型参数 Schema 配置器

### 10.1 结论

v1 不使用纯 JSON 编辑器作为主要编辑方式。

v1 确认提供：

- 表单式参数配置器。

同时支持：

- 高级 JSON 预览。
- JSON 导入。
- JSON 导出。

不允许：

- 要求管理员主要通过手写 JSON 完成模型参数配置。

### 10.2 配置器信息结构

每个参数项至少包含：

- 参数名称。
- 上游字段路径。
- 展示名称。
- 参数类型。
- UI 组件类型。
- 默认值。
- 是否必填。
- 是否高级参数。
- 可选值。
- 最小值。
- 最大值。
- 帮助说明。

### 10.3 参数类型

支持类型：

- `string`
- `number`
- `integer`
- `boolean`
- `enum`
- `array`
- `file`

### 10.4 UI 组件类型

支持组件：

- 文本输入。
- 数字输入。
- 下拉选择。
- 单选按钮。
- 开关。
- 滑块。
- 多选。
- 文件上传。

### 10.5 配置器交互

能力：

- 新增参数。
- 删除参数。
- 拖拽排序。
- 标记高级参数。
- 配置枚举值。
- 配置默认值。
- 校验字段路径。
- 预览前台表单。

校验：

- 参数名称不能为空。
- 上游字段路径不能为空。
- 默认值必须符合类型。
- 枚举参数必须至少有一个选项。
- 最小值不能大于最大值。

## 11. 组件规范

### 11.1 Button

类型：

- Primary：主要动作。
- Secondary：次要动作。
- Ghost：轻量动作。
- Danger：危险动作。

状态：

- default。
- hover。
- disabled。
- loading。

规则：

- 页面主要操作只保留一个 Primary。
- 删除、清空密钥等操作使用 Danger。
- Primary 使用青绿色底和深色文字。
- Secondary 使用暗色 raised surface、边框和浅色文字。

### 11.2 Input

状态：

- default。
- focus。
- error。
- disabled。

规则：

- 密钥输入框默认使用 password 类型。
- 保存后只展示掩码。
- 错误文案放在字段下方。
- 输入框默认使用暗色半透明底，focus 时使用青绿色边框和弱光晕。

### 11.3 Card

用途：

- 模型卡片。
- 任务卡片。
- 结果图卡片。
- 后台导航卡片。

规则：

- 卡片不滥用阴影。
- 工作区卡片优先使用边框。
- 首页展示卡片可以更有层次。
- 默认卡片为暗玻璃面板；重复列表项可使用 `--ds-surface-raised` 或 `--ds-surface-muted` 区分层级。
- 代码块、日志详情和 JSON 预览必须使用暗色代码面板。

### 11.4 Badge

用途：

- 任务状态。
- 模型推荐。
- 支持参考图。
- 配置状态。

状态色：

- 成功：绿色。
- 失败：红色。
- 运行中：琥珀色。
- 等待：中性色。
- 推荐：青绿色。

### 11.5 Dialog

用途：

- 删除确认。
- 批量删除确认。
- 重置密码。
- 清空密钥。
- 查看完整 prompt。
- 查看完整参数。

规则：

- 危险操作按钮靠右。
- 文案明确说明不可恢复或会写审计日志。

### 11.6 Toast

用途：

- 保存成功。
- 测试连接成功。
- 上传成功。
- 任务提交成功。
- 操作失败摘要。

规则：

- 成功 toast 简短。
- 失败 toast 提供下一步。
- 不用 toast 承载长错误详情。

## 12. 状态与空态

### 12.1 Loading

建议：

- 页面初次加载用骨架屏。
- 表单提交用按钮 loading。
- 任务生成中用任务卡片状态，不用全屏 loading。

### 12.2 Empty

需要空态：

- 无模型。
- 无任务。
- 无结果图。
- 无请求日志。
- 无审计日志。

空态规则：

- 告诉用户为什么为空。
- 给出下一步按钮。
- 不使用冷冰冰的“暂无数据”作为唯一文案。

### 12.3 Error

错误状态：

- 表单字段错误。
- 页面级加载错误。
- `new-api` 连接失败。
- 任务失败。
- 存储测试失败。

规则：

- 错误文案要可行动。
- 不直接展示上游原始错误全文。
- 可以提供展开详情给管理员。

---

## 13. 动效规范

### 13.1 推荐动效

- 首页首屏内容 stagger reveal。
- 卡片 hover 轻微上浮。
- 任务状态轻微呼吸或进度感。
- Dialog 淡入和缩放。
- 图片生成成功后结果轻微显现。

### 13.2 避免动效

- 大面积循环动画。
- 按钮过度弹跳。
- 影响输入的动画。
- 长时间阻塞页面的 loading 动画。

### 13.3 性能

规则：

- 动效优先使用 transform 和 opacity。
- 避免频繁 layout 变化。
- 尊重 `prefers-reduced-motion`。

---

## 14. 可访问性

要求：

- 所有表单字段有 label。
- 按钮和链接可键盘聚焦。
- Dialog 打开后焦点进入 Dialog。
- 错误文案与字段关联。
- 颜色不能作为唯一状态表达。
- 图片缩略图需要 alt 或空 alt。
- 主要文字对比度满足 WCAG AA。

---

## 15. shadcn/ui 使用原则

v1 可以使用 shadcn/ui 作为组件基础，但不能直接保留默认模板感。

规则：

- 必须通过 CSS 变量和样式层重塑品牌气质。
- Button、Card、Input、Dialog、Table 需要统一二次封装。
- 不直接在业务页面散落大量原始 shadcn 组件。
- 表格、表单、弹窗统一使用 DreamStudio 风格包装组件。

建议封装：

- `DsButton`
- `DsInput`
- `DsCard`
- `DsDialog`
- `DsDataTable`
- `DsStatusBadge`
- `DsEmptyState`
- `DsFormSection`

## 16. 第七阶段确认项

以下问题已按当前产品方向确认：

- 默认视觉方向采用“近黑中性色 + 青绿色品牌色 + 琥珀强调色”。
- 首页采用“暗色工作室 + 抽象画布 + 任务卡片”的主视觉方向。
- 字体接受标题更有性格、界面保持高可读性的组合。
- 创作台采用左侧 `280-320px` 模型栏 + 右侧统一创作容器；任务列表改为左上角胶囊按钮和浮层。
- 模型参数 Schema 配置器按“表单式配置 + 高级 JSON 预览/导入导出”实现。
