# DreamStudio v1 UI 视觉方向与组件规范

本文档基于 `06-dreamstudio-v1-ia-and-roadmap.md` 编写，用于指导 DreamStudio v1 的视觉风格、页面布局、组件规范、交互状态和前端实现。本文档不替代最终高保真设计稿，但应作为开发初版 UI 的基准。

当前版本状态：确认版。

第七阶段的目标是让 DreamStudio v1 从一开始就具备可信、完整、有创作气质的产品体验，而不是临时后台模板或通用 AI 套壳。

---

## 1. 视觉定位

### 1.1 产品气质

DreamStudio v1 的视觉方向：

```text
专业创作工具 + 温暖工作室 + 轻量未来感
```

关键词：

- 专业。
- 安静。
- 清晰。
- 有创造力。
- 有质感。
- 不压迫。
- 不花哨。

避免：

- 通用 SaaS 后台模板感。
- 过度科技蓝。
- 默认紫白 AI 风格。
- 纯黑暗黑模式。
- 玻璃拟态堆叠。
- 卡片过多导致信息碎片化。

### 1.2 设计目标

v1 UI 需要做到：

- 首次访问时，用户相信这是一个可以认真使用的 AI 创作平台。
- 创作台让用户专注 prompt、模型、参数和结果。
- 管理后台清晰、克制、可靠，不追求花哨。
- 密钥配置流程要降低焦虑，明确告诉用户下一步做什么。
- 任务状态要可感知，避免用户怀疑任务是否丢失。

## 2. 品牌与色彩

### 2.1 色彩方向

确认使用“暖中性色 + 青绿色能量色 + 琥珀强调色”。

主背景：

- `sand-50`：温暖浅米白。
- `stone-950`：深墨灰，用于深色文字。

主品牌色：

- `teal-600`：主要按钮、选中状态、可执行动作。
- `teal-700`：按钮 hover。

辅助强调色：

- `amber-500`：生成中、提示、关键强调。
- `rose-500`：危险操作、失败状态。
- `emerald-600`：成功状态。

中性色：

- `stone-50` 到 `stone-950`。

### 2.2 CSS 变量建议

```css
:root {
  --ds-bg: #f7f1e8;
  --ds-bg-soft: #fbf8f1;
  --ds-surface: #fffdf8;
  --ds-surface-raised: #ffffff;
  --ds-border: #e6dccd;
  --ds-border-strong: #cdbfae;
  --ds-text: #1f1b16;
  --ds-text-muted: #74695d;
  --ds-brand: #0f766e;
  --ds-brand-hover: #115e59;
  --ds-brand-soft: #ccfbf1;
  --ds-accent: #f59e0b;
  --ds-danger: #e11d48;
  --ds-success: #059669;
  --ds-shadow: 0 18px 50px rgba(50, 38, 24, 0.12);
}
```

规则：

- v1 默认使用浅色主题。
- 暗色主题不进入第一验收路径。
- 色彩表达要克制，主界面以中性色和留白为主。
- 生成按钮和关键状态使用品牌色，不要整站大面积铺满品牌色。

## 3. 字体与排版

### 3.1 字体建议

英文字体：

- 标题：`Fraunces` 或 `Newsreader`。
- 正文和界面：`IBM Plex Sans` 或 `Geist`。

中文字体：

- `Noto Serif SC` 可用于首页大标题或品牌语。
- `Noto Sans SC` 可用于界面正文。

说明：

- 不使用 Arial、Roboto、Inter 作为唯一默认视觉。
- 如果工程引入字体成本过高，可优先使用 `Noto Sans SC` + `IBM Plex Sans`。
- 首页标题可以更有性格，后台和创作台保持高可读性。

### 3.2 字号层级

建议层级：

| 用途 | 桌面字号 | 移动字号 |
| --- | --- | --- |
| 首页主标题 | 64px | 40px |
| 页面标题 | 32px | 28px |
| 区块标题 | 20px | 18px |
| 正文 | 15px | 15px |
| 辅助说明 | 13px | 13px |
| 表格文本 | 14px | 14px |

规则：

- 创作台和后台不追求大字号冲击，优先信息密度和清晰度。
- 首页可以使用更大胆的标题排版。
- 表单说明文字要短，不要长篇教育用户。

## 4. 布局系统

### 4.1 页面宽度

建议：

- 首页内容最大宽度：`1200px`。
- 登录注册页表单宽度：`420px`。
- 用户设置页内容宽度：`880px`。
- 管理后台内容最大宽度：`1280px`。
- 创作台桌面端使用全屏工作区。

### 4.2 间距

基础间距：

- `4px` 为最小单位。
- 常用间距：`8px`、`12px`、`16px`、`24px`、`32px`、`48px`。

规则：

- 表单字段之间建议 `16px`。
- 页面区块之间建议 `32px`。
- 管理后台表格密度适中，不要过度稀疏。

### 4.3 圆角与阴影

建议：

- 小控件圆角：`10px`。
- 卡片圆角：`18px`。
- 首页展示卡片圆角：`28px`。
- 弹窗圆角：`20px`。

阴影：

- 工作区内减少重阴影。
- 首页和浮层可使用柔和大阴影。
- 管理后台优先使用边框区分层级。

## 5. 首页设计规范

### 5.1 首页目标

首页 v1 不做临时轻量 Landing，需要具备完整产品表达和可信视觉完成度。

首页要表达：

- DreamStudio 是一个浏览器里的 AI 创作工作室。
- 用户可以通过自己的 `new-api` 密钥使用模型能力。
- DreamStudio 负责更好的创作体验、任务执行和资产管理。
- 用户不需要安装第三方客户端。

### 5.2 首页结构

建议模块：

1. 顶部导航。
2. Hero 主视觉。
3. 创作台能力展示。
4. 使用流程说明。
5. 任务和资产管理能力说明。
6. 管理员可配置能力说明。
7. 底部 CTA。

### 5.3 Hero

Hero 内容：

- 主标题：强调“一站式 AI 图片创作工作室”。
- 副标题：说明对接用户自己的 `new-api` 密钥。
- 主按钮：开始创作。
- 次按钮：登录。
- 轻提示：使用前需要准备 `new-api` API 密钥。

视觉建议：

- 使用温暖渐变背景。
- 使用抽象画布、生成结果网格、任务队列卡片构成主视觉。
- 不直接使用厂商 Logo 堆砌。
- 不用夸张赛博风。

主视觉结论：

- v1 首页采用“温暖工作室 + 抽象画布 + 任务卡片”的主视觉方向。

### 5.4 首页背景

建议：

- 米色到浅青绿色的柔和渐变。
- 叠加极淡噪点或纸张纹理。
- 使用 1 到 2 个抽象光斑，不要满屏光污染。

### 5.5 首页动效

推荐：

- 首屏标题和卡片轻微错位进入。
- 任务卡片状态可做轻微脉冲。
- 结果图网格可做慢速浮动。

避免：

- 大量滚动视差。
- 复杂 3D。
- 动效影响表单可用性。

## 6. 登录、注册与密钥引导

### 6.1 登录注册页

视觉目标：

- 安静。
- 可信。
- 不像临时后台登录框。

布局：

- 左侧品牌说明或创作插画。
- 右侧登录/注册表单。
- 移动端只保留表单和简短品牌语。

状态：

- 表单错误使用明确文案。
- 登录中按钮展示 loading。
- 被禁用用户展示独立提示。

### 6.2 密钥配置引导

密钥配置是用户的第一个关键门槛，需要尽量降低焦虑。

页面结构：

- 标题：连接你的 `new-api` 密钥。
- 三步说明：前往 `new-api`、创建密钥、粘贴到 DreamStudio。
- 默认服务地址展示。
- API 密钥输入。
- 测试连接。
- 保存并进入创作台。

文案规则：

- 明确说明 DreamStudio 不处理充值和额度。
- 明确说明密钥会加密保存。
- 不展示密钥明文。
- 测试失败时给出下一步建议。

状态设计：

- 未测试：中性提示。
- 测试中：按钮 loading，表单禁用。
- 测试成功：绿色成功状态。
- 测试失败：红色错误摘要 + 修复建议。

## 7. AI 创作台设计规范

### 7.1 桌面三栏布局

v1 AI 创作台采用：

```text
左侧模型栏 | 中间创作栏 | 右侧任务与结果栏
```

确认宽度：

- 左侧模型栏：`280px` 到 `320px`。
- 中间创作栏：自适应，最小 `520px`。
- 右侧任务栏：`360px` 到 `420px`。

布局规则：

- 三栏高度占满视口。
- 左右栏可独立滚动。
- 中间创作栏优先保证 prompt 和生成按钮可见。
- 生成按钮建议固定在中间栏底部或参数面板底部。

### 7.2 左侧模型栏

组件：

- 分类 tabs。
- 推荐模型区。
- 模型列表。
- 模型状态标签。

模型卡片展示：

- 展示名称。
- 厂商名称。
- 是否支持参考图。
- 推荐标签。
- 简短能力说明。

状态：

- 选中。
- hover。
- 禁用。
- 推荐。

### 7.3 中间创作栏

组件：

- Prompt 编辑器。
- 负面提示词，高级区默认折叠。
- 参考图上传区。
- 参数表单。
- 生成按钮。

Prompt 编辑器：

- 支持多行输入。
- 高度不低于 `160px`。
- 提供字数或 token 近似提示，但不强制精确。
- 为空时生成按钮禁用。

参考图上传：

- 不支持参考图的模型隐藏或禁用上传区。
- 上传后展示缩略图。
- 可删除已上传参考图。

参数表单：

- 基础参数直接展示。
- 高级参数折叠。
- 不兼容参数不展示。
- 使用模型 `parameter_schema` 驱动。

### 7.4 右侧任务与结果栏

组件：

- 当前任务卡片。
- 最近任务列表。
- 结果图预览网格。
- 失败摘要。
- 下载入口。

状态展示：

- `pending`：等待中。
- `running`：生成中。
- `succeeded`：已完成。
- `failed`：失败。
- `timeout`：超时。
- `canceled`：已取消。

规则：

- `pending` 展示取消按钮。
- `running` 展示不可取消提示。
- 成功后优先展示结果图。
- 失败时展示可读错误和重试按钮。

### 7.5 移动端布局

移动端降级：

- 顶部模型选择。
- 中间 prompt 和参数。
- 下方任务和结果。
- 模型列表可用抽屉。

规则：

- 移动端不强制三栏同屏。
- 生成按钮应始终容易触达。
- 任务结果不要被参数表单淹没。

## 8. 资产仓库设计规范

### 8.1 范围

v1 资产仓库只展示结果图。

不展示：

- 参考图列表。
- 文件夹。
- 标签。
- 分享链接。
- 回收站。

说明：

- 参考图仍保存为资产，用于任务执行和任务详情展示。
- 用户不在资产仓库中独立管理参考图。

### 8.2 布局

推荐：

- 顶部筛选区。
- 瀑布流或规则网格。
- 右侧或弹窗预览详情。

卡片展示：

- 图片缩略图。
- 模型名称。
- 创建时间。
- 下载按钮。
- 删除按钮。

批量选择：

- 进入选择模式后显示多选框。
- 批量删除前二次确认。

### 8.3 空状态

文案示例：

```text
还没有生成结果。
去创作台生成第一张作品。
```

空状态应提供进入 `/studio` 的按钮。

## 9. 管理后台设计规范

### 9.1 后台首页

v1 后台首页只做导航面板和关键配置入口。

模块：

- 用户管理入口。
- 模型管理入口。
- 系统设置入口。
- 存储设置入口。
- 请求日志入口。
- 审计日志入口。

不做：

- 复杂统计图表。
- 实时大屏。
- 成本分析。
- 收入分析。

### 9.2 后台整体风格

后台应：

- 信息密度适中。
- 表格清晰。
- 操作按钮明确。
- 危险操作需要二次确认。
- 以可靠为主，不追求强视觉冲击。

### 9.3 表格

表格规范：

- 每页默认 20 条。
- 支持筛选和分页。
- 行操作放在右侧。
- 危险操作使用红色或二次确认。

适用页面：

- 用户管理。
- 模型管理。
- 模型候选快照。
- 请求日志。
- 审计日志。

### 9.4 表单

表单规范：

- 字段标签清晰。
- 帮助文本短且具体。
- 保存按钮固定在表单底部或右上角。
- 保存成功有 toast。
- 保存失败展示错误摘要。

适用页面：

- 系统设置。
- 存储设置。
- 模型编辑。
- 用户密钥代配置。

## 10. 模型参数 Schema 配置器

### 10.1 结论

v1 不使用纯 JSON 编辑器作为主要编辑方式。

v1 确认提供：

- 表单式参数配置器。

同时支持：

- 高级 JSON 预览。
- JSON 导入。
- JSON 导出。

不允许：

- 要求管理员主要通过手写 JSON 完成模型参数配置。

### 10.2 配置器信息结构

每个参数项至少包含：

- 参数名称。
- 上游字段路径。
- 展示名称。
- 参数类型。
- UI 组件类型。
- 默认值。
- 是否必填。
- 是否高级参数。
- 可选值。
- 最小值。
- 最大值。
- 帮助说明。

### 10.3 参数类型

支持类型：

- `string`
- `number`
- `integer`
- `boolean`
- `enum`
- `array`
- `file`

### 10.4 UI 组件类型

支持组件：

- 文本输入。
- 数字输入。
- 下拉选择。
- 单选按钮。
- 开关。
- 滑块。
- 多选。
- 文件上传。

### 10.5 配置器交互

能力：

- 新增参数。
- 删除参数。
- 拖拽排序。
- 标记高级参数。
- 配置枚举值。
- 配置默认值。
- 校验字段路径。
- 预览前台表单。

校验：

- 参数名称不能为空。
- 上游字段路径不能为空。
- 默认值必须符合类型。
- 枚举参数必须至少有一个选项。
- 最小值不能大于最大值。

## 11. 组件规范

### 11.1 Button

类型：

- Primary：主要动作。
- Secondary：次要动作。
- Ghost：轻量动作。
- Danger：危险动作。

状态：

- default。
- hover。
- disabled。
- loading。

规则：

- 页面主要操作只保留一个 Primary。
- 删除、清空密钥等操作使用 Danger。

### 11.2 Input

状态：

- default。
- focus。
- error。
- disabled。

规则：

- 密钥输入框默认使用 password 类型。
- 保存后只展示掩码。
- 错误文案放在字段下方。

### 11.3 Card

用途：

- 模型卡片。
- 任务卡片。
- 结果图卡片。
- 后台导航卡片。

规则：

- 卡片不滥用阴影。
- 工作区卡片优先使用边框。
- 首页展示卡片可以更有层次。

### 11.4 Badge

用途：

- 任务状态。
- 模型推荐。
- 支持参考图。
- 配置状态。

状态色：

- 成功：绿色。
- 失败：红色。
- 运行中：琥珀色。
- 等待：中性色。
- 推荐：青绿色。

### 11.5 Dialog

用途：

- 删除确认。
- 批量删除确认。
- 重置密码。
- 清空密钥。
- 查看完整 prompt。
- 查看完整参数。

规则：

- 危险操作按钮靠右。
- 文案明确说明不可恢复或会写审计日志。

### 11.6 Toast

用途：

- 保存成功。
- 测试连接成功。
- 上传成功。
- 任务提交成功。
- 操作失败摘要。

规则：

- 成功 toast 简短。
- 失败 toast 提供下一步。
- 不用 toast 承载长错误详情。

## 12. 状态与空态

### 12.1 Loading

建议：

- 页面初次加载用骨架屏。
- 表单提交用按钮 loading。
- 任务生成中用任务卡片状态，不用全屏 loading。

### 12.2 Empty

需要空态：

- 无模型。
- 无任务。
- 无结果图。
- 无请求日志。
- 无审计日志。

空态规则：

- 告诉用户为什么为空。
- 给出下一步按钮。
- 不使用冷冰冰的“暂无数据”作为唯一文案。

### 12.3 Error

错误状态：

- 表单字段错误。
- 页面级加载错误。
- `new-api` 连接失败。
- 任务失败。
- 存储测试失败。

规则：

- 错误文案要可行动。
- 不直接展示上游原始错误全文。
- 可以提供展开详情给管理员。

## 13. 动效规范

### 13.1 推荐动效

- 首页首屏内容 stagger reveal。
- 卡片 hover 轻微上浮。
- 任务状态轻微呼吸或进度感。
- Dialog 淡入和缩放。
- 图片生成成功后结果轻微显现。

### 13.2 避免动效

- 大面积循环动画。
- 按钮过度弹跳。
- 影响输入的动画。
- 长时间阻塞页面的 loading 动画。

### 13.3 性能

规则：

- 动效优先使用 transform 和 opacity。
- 避免频繁 layout 变化。
- 尊重 `prefers-reduced-motion`。

## 14. 可访问性

要求：

- 所有表单字段有 label。
- 按钮和链接可键盘聚焦。
- Dialog 打开后焦点进入 Dialog。
- 错误文案与字段关联。
- 颜色不能作为唯一状态表达。
- 图片缩略图需要 alt 或空 alt。
- 主要文字对比度满足 WCAG AA。

---

## 15. shadcn/ui 使用原则

v1 可以使用 shadcn/ui 作为组件基础，但不能直接保留默认模板感。

规则：

- 必须通过 CSS 变量和样式层重塑品牌气质。
- Button、Card、Input、Dialog、Table 需要统一二次封装。
- 不直接在业务页面散落大量原始 shadcn 组件。
- 表格、表单、弹窗统一使用 DreamStudio 风格包装组件。

建议封装：

- `DsButton`
- `DsInput`
- `DsCard`
- `DsDialog`
- `DsDataTable`
- `DsStatusBadge`
- `DsEmptyState`
- `DsFormSection`

## 16. 第七阶段确认项

以下问题已按当前产品方向确认：

- 默认视觉方向采用“暖中性色 + 青绿色品牌色 + 琥珀强调色”。
- 首页采用“温暖工作室 + 抽象画布 + 任务卡片”的主视觉方向。
- 字体接受标题更有性格、界面保持高可读性的组合。
- 创作台三栏宽度采用左 `280-320px`、右 `360-420px` 的范围。
- 模型参数 Schema 配置器按“表单式配置 + 高级 JSON 预览/导入导出”实现。
